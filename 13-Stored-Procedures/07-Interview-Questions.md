## Scenario-Based Questions

### 21. You need to execute the same business logic thousands of times daily. What would you use?

Answer:

Stored Procedure

Reason:

Reusability and Performance.

---

### 22. You need secure access to database operations without exposing table structures. What would you use?

Answer:

Stored Procedure

Reason:

Security and Access Control.

---

### 23. An application frequently performs money transfers between accounts. How can Stored Procedures help?

Answer:

Encapsulate transaction logic inside a procedure for consistency and reliability.

---

### 24. Why do enterprise applications heavily use Stored Procedures?

Answer:

- Faster Execution
- Reduced Network Traffic
- Security
- Centralized Business Logic

---

### 25. Difference between Trigger, Function, and Stored Procedure?

Answer:

Trigger:
Automatically executes on events.

Function:
Returns a value.

Stored Procedure:
Executes a set of SQL statements and business logic.
