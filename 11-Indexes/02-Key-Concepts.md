# 🔑 Key Concepts

## Primary Index

Created automatically on Primary Key.

Example:

StudentID

---

## Secondary Index

Created on non-primary attributes.

Example:

Name

---

## Clustered Index

Determines physical order of data.

Only one clustered index per table.

---

## Non-Clustered Index

Separate structure pointing to data.

Multiple allowed.

---

## Composite Index

Created on multiple columns.

Example:

(Name, Age)

---

## Unique Index

Prevents duplicate values.

---

## B-Tree Index

Most commonly used index structure.
