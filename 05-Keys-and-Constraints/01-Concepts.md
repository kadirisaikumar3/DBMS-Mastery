# 📖 Keys and Constraints

## What is a Key?

A Key is an attribute or set of attributes used to identify records uniquely.

Example:

Student Table

| ID | Name |
|----|------|
| 1 | Sai |
| 2 | John |

ID uniquely identifies students.

---

## Why Keys?

Keys help:

- Uniquely identify records
- Avoid duplication
- Establish relationships
- Improve data integrity

---

## Types of Keys

1. Super Key
2. Candidate Key
3. Primary Key
4. Alternate Key
5. Composite Key
6. Foreign Key

---

## What are Constraints?

Constraints are rules applied on database columns.

They ensure:

- Accuracy
- Consistency
- Integrity

Examples:

- NOT NULL
- UNIQUE
- CHECK
- DEFAULT
