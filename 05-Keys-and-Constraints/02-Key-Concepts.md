# 🔑 Key Concepts

## Super Key

Set of attributes that uniquely identifies records.

Example:

StudentID

StudentID + Name

---

## Candidate Key

Minimal Super Key.

Example:

StudentID

Email

---

## Primary Key

Chosen Candidate Key.

Properties:

- Unique
- Not NULL

---

## Alternate Key

Candidate Key not selected as Primary Key.

---

## Composite Key

Combination of multiple columns.

Example:

StudentID + CourseID

---

## Foreign Key

Creates relationship between tables.

Example:

StudentID in Orders Table.

---

## Constraints

Rules enforced on columns.

Examples:

NOT NULL

UNIQUE

CHECK

DEFAULT
