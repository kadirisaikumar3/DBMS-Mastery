## Vertical Scaling

Increase CPU

Increase RAM

Increase Storage

One Server

---

## Horizontal Scaling

Add More Servers

Distribute Load

Better Scalability

## Sharding

Split database into smaller parts.

Each server stores part of data.

---

## Replication

Maintain multiple copies of data.

## Read Replica

Used only for read queries.

Reduces primary database load.

## Load Balancer

Distributes database traffic.
