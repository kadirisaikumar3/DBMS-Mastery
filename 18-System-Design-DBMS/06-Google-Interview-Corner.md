# 🎯 Google Interview Corner

## Most Asked Concepts

### Vertical Scaling

Increase:

- CPU
- RAM
- Storage

Single Machine

---

### Horizontal Scaling

Add More Servers

↓

Better Scalability

---

### Sharding

Split data across servers.

Example:

Users A-M

↓

Server 1

Users N-Z

↓

Server 2

---

### Replication

Primary Database

↓

Replica Database

Purpose:

- High Availability
- Disaster Recovery

---

### Read Replicas

Handle Read Queries.

Primary handles Writes.

---

### Redis Cache

Stores frequently accessed data in memory.

Benefits:

- Faster Response Time
- Reduced Database Load

---

## Interview Tip

Remember:

Scaling

↓

Replication

↓

Caching

↓

Load Balancing

↓

High Availability

This sequence is commonly discussed in system design interviews.
