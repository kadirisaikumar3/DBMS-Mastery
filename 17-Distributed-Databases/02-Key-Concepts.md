# 🔑 Key Concepts

## Data Fragmentation

Splitting a database into smaller fragments.

Types:

- Horizontal Fragmentation
- Vertical Fragmentation
- Hybrid Fragmentation

---

## Data Replication

Maintaining copies of data at multiple locations.

Benefits:

- High Availability
- Faster Reads
- Fault Tolerance

---

## Distributed Transactions

Transactions spanning multiple database nodes.

---

## Two-Phase Commit (2PC)

Phase 1:

Prepare

↓

Phase 2:

Commit

or

Rollback

---

## Three-Phase Commit (3PC)

Improved version of 2PC that reduces blocking.

---

## Distributed Query Processing

Optimizes queries across multiple database servers.
